🔧 PROBLEMA IDENTIFICADO
========================
Los botones con onclick inline se ejecutaban ANTES de que
el DOMContentLoaded registrara las funciones en window.
Resultado: onclick="startTest()" no encontraba la función.

💡 SOLUCIÓN IMPLEMENTADA
========================
✅ Eliminados TODOS los onclick/onsubmit inline del HTML
✅ Agregados IDs únicos a cada botón
✅ Implementado setupEventListeners() en script.js
✅ Event listeners se asignan DESPUÉS de DOMContentLoaded
✅ Logs detallados para cada paso

🎯 BOTONES CORREGIDOS
=====================
✅ startTestBtn      → Comenzar Test (PRINCIPAL)
✅ userForm          → Submit formulario
✅ goBackBtn         → Volver al inicio
✅ exitTestBtn       → Salir del test
✅ retakeTestBtn     → Realizar de nuevo
✅ downloadResultsBtn → Descargar PDF

📝 CAMBIOS EN LOS ARCHIVOS
===========================

index.html:
-----------
ANTES: <button onclick="startTest()">
AHORA: <button id="startTestBtn">

ANTES: <form onsubmit="submitUserForm(event)">
AHORA: <form id="userForm">

script.js:
----------
AGREGADO: función setupEventListeners()
AGREGADO: Logs de "🖱️ Click en botón..."
AGREGADO: Verificación de elementos encontrados
MEJORADO: Manejo de errores más robusto

🐛 SI ALGO FALLA
================

1. NO VEO "Event listener agregado: startTestBtn"
   → Recarga con Ctrl+F5 (limpia caché)
   → Verifica que script.js esté en el servidor

2. NO VEO "Click en botón Comenzar Test" al hacer clic
   → El event listener no se agregó
   → Recarga la página completamente
   → Verifica en consola si hay errores en rojo

3. VEO el click pero no se muestra el formulario
   → Revisa si hay errores en ROJO en consola
   → Puede ser problema con appState
   → Recarga con Ctrl+F5

4. LA CONSOLA ESTÁ VACÍA
   → script.js no se cargó
   → Verifica la ruta: <script src="script.js"></script>
   → Abre pestaña Network en F12, verifica que se descargue
